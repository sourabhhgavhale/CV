OS notes :

  1. Deadlocks

    2. Necessary Conditions for Deadlock :

    1. Mutual exclusion
      2. One resource is held in nonshareble mode. If one process is holding resouce and another resource request it then requesting process should be delayed until it is released.
    2. Hold and wait :
      3. process is holding atleast one resource and waiting to aquire additional resource held by another process.
    3. No preemtion :
      4. Resource cant be preempted. Resource can be released only after the process holding it releases the resource after complition.
    4. Circular wait :
      5. A set of process exists P ={p1,p2,p2,...pn} such that p1 is waiting for resource held by p2, p2 is watining for resource held by p3 and so no.. pn is waiting on resouce held by p0.

    Resouce Allocation Graph :

    If there is no cycle in the resource allocation graph of the processes then there is no deadlock. But if cycle is present in the graph then deadlock may be the present.

    Deadlocks prevention :

    By ensuring atleast one of the conditions responsible for deadlock doesnt hold we can prevent deadlock occurence.
    1. Mutual exclusion
      2. The mutual-exclusion condition must hold for nonsharable resources.
    2. Hold and wait :
      3. To ensure that the hold-and-wait condition never occurs in the system, we must guarantee that, whenever a process requests a resource, it does not hold any other resources.
    3. No preemtion :
      4. If a process is holding some resources and requests another resource that cannot be immediately allocated to it (that is, the process must wait), then all resources the process is currently holding are preempted. In other words, these resources are implicitly released. The preempted resources are added to the list of resources for which the process is waiting. The process will be restarted only when it can regain its old resources, as well as the new ones that it is requesting
    4. Circular wait :
      5. One way to ensure that this condition never holds is to impose a total ordering of all resource types and to require that each process requests resources in an increasing order of enumeration.
